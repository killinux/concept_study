# 用 Claude 提升调研与学习效率

## 一、调研阶段 — 主动式文献助手

### 1.1 概念拆解

把大主题交给 Claude 拆成知识树，快速建立全局视野。

**常用 Prompt 模式：**

- "X 的核心前提假设是什么？违反这些假设时分别退化为什么问题？"
- "X 方法和 Y 方法的本质区别是什么？用一个直觉类比解释"
- "这个领域的经典问题有哪些？按难度排序"

**示例（运筹学场景）：**

> "线性规划的五个核心假设（比例性、可加性、确定性、可分性、非负性）分别在什么现实场景下会被违反？违反后应该用什么替代模型？"

### 1.2 对比矩阵

让 Claude 生成结构化对比表，快速看清概念间的关系。

**Prompt 模板：**

> "用表格对比 A、B、C 三个方法，维度包括：适用场景、计算复杂度、核心假设、典型算法、局限性"

### 1.3 追问链

读完教材或论文后，用追问链挖掘隐含信息。

**追问方向：**

| 方向 | Prompt |
|------|--------|
| 隐含假设 | "这段推导隐含了哪些未明说的假设？" |
| 反例 | "这个结论在什么条件下不成立？举一个反例" |
| 联系 | "这个概念和 X 有什么关系？它们的统一框架是什么？" |
| 历史 | "这个方法是为了解决什么历史问题而被发明的？" |

## 二、学习阶段 — 费曼学习法搭档

### 2.1 反向教学

向 Claude 讲解你刚学的概念，让它挑出理解偏差。

**Prompt：**

> "我现在给你讲 [概念名]，你听完后：1）指出我理解不准确的地方；2）补充我遗漏的关键点；3）问我一个检验理解深度的问题"

### 2.2 生成练习题

让 Claude 按难度梯度出题，先做再看答案。

**Prompt：**

> "根据 [主题]，出 3 道题：第 1 道考基本概念，第 2 道考应用，第 3 道考边界情况。先不给答案，等我做完再逐题讲解"

### 2.3 错误归因分析

做错题后，不只看正确答案，而是定位思路偏差点。

**Prompt：**

> "我的解题过程是 [你的步骤]。请不要直接给答案，而是告诉我从第几步开始出了问题，以及错误的根本原因是什么"

## 三、输出阶段 — 知识沉淀加速

### 3.1 笔记审查

让 Claude 检查已有笔记的逻辑完整性和知识覆盖。

**Prompt：**

> "读一下我的笔记，检查：1）哪些概念之间缺了过渡？2）哪些定义不够严谨？3）建议补充什么内容？"

### 3.2 代码验证概念

把抽象概念变成可运行的代码，加深直觉理解。

**Prompt：**

> "用 Python 实现 [算法名] 的核心步骤，每一步打印中间状态。用一个小规模例子演示完整过程"

**示例：**

```python
# 用代码验证单纯形法的每一步
# 输入：目标函数系数、约束矩阵、右端向量
# 输出：每次迭代的 tableau、进基/出基变量、目标函数值变化
```

### 3.3 周期性回顾

每学完一个模块，生成自测清单。

**Prompt：**

> "针对 [模块名]，列出 10 个我应该能回答的问题，按'记忆→理解→应用'三个层次排列"

---

## 四、核心原则

```
先自己想 → 和 Claude 对话 → 自己总结
```

- **先想再问**：自己先形成初步理解，再用 Claude 验证和补充，效率远高于直接问答
- **追求理解而非答案**：让 Claude 解释"为什么"，而不是直接给"是什么"
- **主动构建联系**：每学一个新概念，让 Claude 帮你和已有知识建立关联
- **定期回顾**：利用 Claude 生成的自测题做间隔重复，对抗遗忘曲线

## 五、在 Claude Code 中的实践技巧

| 技巧 | 说明 |
|------|------|
| 贴入笔记让 Claude 审查 | 直接在对话中贴 Markdown 笔记，让 Claude 逐段点评 |
| 用代码验证数学概念 | 请 Claude 写可运行脚本，用具体数值验证抽象定理 |
| 生成对比文档 | 让 Claude 输出 Markdown 表格，直接存入笔记体系 |
| 迭代式追问 | 一个概念追问 3-5 轮，比问 5 个不同概念收获更大 |
| 让 Claude 读你的笔记目录 | 它可以发现你的知识盲区并建议下一步学什么 |
